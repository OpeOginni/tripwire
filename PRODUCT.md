# Product

## Register

product

## Users

Open source maintainers and teams managing GitHub organizations. Ranges from solo maintainers with a handful of repos to teams running large orgs with heavy inbound activity.

Usage is both reactive (responding to spam alerts, suspicious PRs, flagged contributors) and proactive (checking repo health, reviewing reputation scores, tuning moderation rules). The tool runs in the background and surfaces what matters.

## Product Purpose

Tripwire is a GitHub moderation platform. It catches low-signal activity — spam PRs, bot accounts, suspicious contributors — before it wastes maintainer time. AI-powered detection, configurable rules, contributor reputation scoring, and a built-in chat interface for managing everything through natural language.

Success looks like: maintainers trust Tripwire to handle the noise so they can focus on real contributions.

## Brand Personality

Stark. Minimal. Effective.

No warmth, no friendliness, no hand-holding. Tripwire is a tool that watches and acts. The surveillance-eye branding is intentional — this is a sentinel, not an assistant. Confidence comes from restraint: show only what matters, say only what's needed.

## Anti-references

- **Linear** — too polished, too smooth, too "design tool for designers." Tripwire is rougher, more utilitarian.
- **Better Auth** — generic SaaS template energy. Nothing distinctive.
- **Generic SaaS dashboards** — cream backgrounds, friendly illustrations, rounded everything. None of that.
- **Busy monitoring UIs** (Datadog, Grafana) — information density without hierarchy. Tripwire is selective about what it surfaces.

## Design Principles

1. **Surveillance, not assistance.** Tripwire watches. The interface should feel like a control room, not a help desk. Information is presented as intelligence, not suggestions.
2. **Silence is a feature.** Empty states and quiet moments mean the system is working. Don't fill space to fill space.
3. **Earned attention.** Every element on screen must justify its presence. If it doesn't demand action or convey signal, it doesn't belong.
4. **Raw over refined.** Prefer the slightly rough, the monospaced, the terminal-adjacent. Polish exists in precision, not decoration.
5. **The vibes are load-bearing.** The retro-terminal, CRT, surveillance aesthetic isn't skin — it's identity. Design decisions should reinforce this tone, not soften it.

## Accessibility & Inclusion

WCAG AA minimum. Dark theme is primary and permanent — ensure sufficient contrast ratios against the charcoal palette. Support reduced motion preferences for the landing page's heavier animations (3D terminal, Space Invaders, CRT effects). Keyboard navigation throughout the dashboard.
